# Οδηγός Ανεβάσματος στο Internet (GitHub Pages & Papaki)

Αυτός ο οδηγός θα σε βοηθήσει να ανεβάσεις το site σου δωρεάν στο GitHub Pages και να συνδέσεις το domain `nikoskoufos.gr` που έχεις στο Papaki.

## Βήμα 1: Δημιουργία Λογαριασμού & Repository στο GitHub

1. Πήγαινε στο [github.com](https://github.com) και κάνε **Sign up** (αν δεν έχεις ήδη λογαριασμό).
2. Αφού συνδεθείς, πάτα το σύμβολο **+** πάνω δεξιά και επίλεξε **New repository**.
3. Στο "Repository name", δώσε ένα όνομα (π.χ. `patisserie-nikos-koufos`).
4. Πάτα **Create repository**.

## Βήμα 2: Ανέβασμα Αρχείων

1. Στη σελίδα του repository που μόλις έφτιαξες, πάτα το link **uploading an existing file**.
2. Σύρε **ΟΛΑ** τα αρχεία από τον φάκελο του project σου (`index.html`, `style.css`, `script.js`, και τον φάκελο `assets`) μέσα στο γκρι πλαίσιο.
   - *Προσοχή:* Βεβαιώσου ότι το `index.html` είναι χύμα στον κεντρικό φάκελο, όχι μέσα σε υποφάκελο.
3. Περίμενε να φορτώσουν όλα τα αρχεία.
4. Κάτω χαμηλά, στο προτεινόμενο κείμενο "Add files via upload", πάτα το πράσινο κουμπί **Commit changes**.

## Βήμα 3: Ενεργοποίηση Site (GitHub Pages)

1. Στο μενού του repository (πάνω), πήγαινε στα **Settings**.
2. Στο αριστερό μενού, βρες και πάτα την ενότητα **Pages**.
3. Στο πεδίο **Source**, άλλαξε το από "None" σε **Deploy from a branch**.
4. Στο **Branch**, επίλεξε `main` (ή `master`) και φάκελο `/ (root)`.
5. Πάτα **Save**.
6. Περίμενε 1-2 λεπτά και κάνε ανανέωση στη σελίδα. Θα δεις ένα μήνυμα "Your site is live at..." με ένα link.

## Βήμα 4: Σύνδεση με το Domain σου (Papaki)

### Στο GitHub:
1. Στην ίδια σελίδα (**Settings** -> **Pages**).
2. Στο πεδίο **Custom domain**, γράψε `www.nikoskoufos.gr` (ή `nikoskoufos.gr`).
3. Πάτα **Save**. Θα ξεκινήσει ένας έλεγχος DNS.

### Στο Papaki.gr:
1. Συνδέσου στον λογαριασμό σου στο Papaki.
2. Πήγαινε σταdomains σου και επίλεξε το `nikoskoufos.gr`.
3. Πήγαινε στην επεξεργασία της **Υπηρεσίας DNS** (αν δεν είναι ενεργοποιημένη, ενεργοποίησέ τη δωρεάν - Basic DNS).
4. Πρόσθεσε τις εξής εγγραφές (Records):

   **Εγγραφή 1 (για το www):**
   - **Τύπος (Type):** `CNAME`
   - **Host / Name:** `www`
   - **Value / Target:** `ΤΟ_USERNAME_ΣΟΥ.github.io` (π.χ. `sgooman.github.io` - δες το link από το Βήμα 3).

   **Εγγραφή 2 (για το σκέτο domain):**
   - **Τύπος (Type):** `A`
   - **Host / Name:** `@` (ή κενό, ανάλογα πώς το ζητάει το Papaki)
   - **Value / Target:** `185.199.108.153`

   **Εγγραφή 3 (προαιρετικά, για ασφάλεια):**
   - **Τύπος (Type):** `A`
   - **Host / Name:** `@`
   - **Value / Target:** `185.199.109.153`

5. Αποθήκευσε τις αλλαγές.

## Βήμα 5: HTTPS (Το Λουκετάκι Ασφαλείας)

1. Γύρισε στο GitHub -> Settings -> Pages.
2. Αφού περάσει λίγη ώρα και το GitHub δει τις αλλαγές στο Papaki, θα σου επιτρέψει να τσεκάρεις το κουτάκι **Enforce HTTPS**.
3. Τσέκαρέ το για να είναι το site ασφαλές.

### Συχνές Ερωτήσεις (FAQ)

**Θα πέσει το site μου κατά τη διάρκεια των αλλαγών;**
- **Όχι ακριβώς.** Το site στο GitHub (το λινκ `username.github.io`) θα δουλεύει **πάντα**.
- Το δικό σου domain (`nikoskoufos.gr`) μπορεί να θέλει από **1 έως 48 ώρες** για να "δει" τις νέες ρυθμίσεις σε όλο τον κόσμο.
- Σε αυτό το διάστημα, κάποιοι μπορεί να βλέπουν το νέο site, κάποιοι το παλιό, ή κάποιοι μια προσωρινή σελίδα λάθους.
- Είναι φυσιολογικό! Μην ανησυχήσεις αν δεν παίξει αμέσως στο κινητό σου ή στον υπολογιστή σου. Δοκίμασε από άλλο δίκτυο (π.χ. 4G vs WiFi) ή περίμενε λίγο.
